## 2019년 1학기 네트워크 프로그래밍 숙제 2 [전체성적의 10%]

출제: 5월 2일, 제출마감: 5월 16일

 

교재의 MulticastSender와 MulticastReceiver를 통합하여 대화상자(Dialog)기반 다자간 채팅프로그램을 구성하시오. (수업시간에 설명한 multichat.c를 수정하여 작성하여도 됨)

1. [3점] 멀티캐스트 주소와 포트를 임의로 입력을 하고 대화명을 입력하여 채팅에 참여할 수 있도록 구성하시오.

- 주소에 Class D 가 아닌 IP 주소를 입력하면 오류메시지를 출력하고, 재입력하도록 구성하기 바람. (1점)

- 포트에 적합한 데이터가 아닌 것 (한글이나 영문자, Special Character, 포트번호로 줄 수 없는 숫자의 범위)에 대하여 오류메시지를 출력하고 재입력하도록 구성하기 바람. (1점)

- 대화명을 바꾸고 싶을때는 언제라도 변경할 수 있도록 구성해야 함. 이경우는 구 대화명과 신 대화명이 바뀜이 자동으로 채팅창에 시간과 함께 출력 되어야 함. (1점)

- 채팅 메시지가 오가지 않는 상황에서 화면만 구성한 경우는 0 점임.

2. [1점] 채팅메시지에는 대화명과 IP주소 (멀티캐스트 주소가 아니라 호스트 주소), 대화가 출력되는 시간이 포함되어야 함.

- 대화명과 IP주소, 출력시간, 채팅 메시지가 출력되는 형태임. (1점)

추가적인 정보를 출력하여도 됨. (Hint: 아래 3번을 해결하기 위한 정보가 있으면 추가로 출력하면 됨)

3. [4점] 채팅에 접속한후 채팅창에 나의 대화명과 같은 대화명이 있는 메시지가 들어오면 자동으로 경고 메시지를 보내도록 하시오. (3점)

- 경고메시지는 상대방 사용자가 대화명을 바꾸지 않고 메시지를 보내오면 다시 보냄.

- 경고메시지의 형태는 자유로이 구성하기 바람.

- 경고메시지를 먼저 받은 사용자는 대화명이 같은 상대방에게 경고 메시지를 보낼수 없음. 즉 대화명을 바꾸어야 경고메시지를 안 받음.

4. [2점] 어떻게 프로그램을 설계하였는지와 실행 예에 대하여 리포트를 작성하시오.
